리스트 컴프리헨션 List Comprehension

08 / 17
for 루프 방식 (기존)
# 각 숫자의 제곱을 리스트에 담기 squares = [] for x in range(1, 6): squares.append(x ** 2) # → [1, 4, 9, 16, 25] # 짝수만 골라 담기 evens = [] for x in range(10): if x % 2 == 0: evens.append(x) # → [0, 2, 4, 6, 8]
→
리스트 컴프리헨션
# 구조 [표현식 for 변수 in 반복가능객체] [표현식 for 변수 in 반복가능객체 if 조건] # 제곱 리스트 squares = [x ** 2 for x in range(1, 6)] # → [1, 4, 9, 16, 25] # 조건 필터링 evens = [x for x in range(10) if x % 2 == 0] # → [0, 2, 4, 6, 8]
변환 + 필터 동시에
words = ['hi', 'hello', 'hey'] # 3글자 초과인 단어만 대문자로 result = [w.upper() for w in words if len(w) > 2] # → ['HELLO', 'HEY'] # 문자열 리스트 → 숫자 리스트 nums = [int(x) for x in ['1','2','3']] # → [1, 2, 3]
딕셔너리 · 셋 컴프리헨션
# 딕셔너리 컴프리헨션 { k: v for ... } sq_d = {x: x**2 for x in range(1, 4)} # → {1: 1, 2: 4, 3: 9} # 키-값 뒤집기 d = {'a':1, 'b':2} inv = {v: k for k, v in d.items()} # → {1: 'a', 2: 'b'} # 셋 컴프리헨션 { 표현식 for ... } unique = {x % 3 for x in range(9)} # → {0, 1, 2}
💡 언제 쓸까? — 간단한 변환·필터는 컴프리헨션이 간결합니다. 로직이 복잡하거나 2단계 이상 중첩된다면 일반 for 루프가 더 읽기 쉽습니다.

홀수(i%2 != 0)만 골라서 i*i(제곱)을 리스트에 추가합니다. 1~5 중 홀수는 1, 3, 5.
홀수: 1→1², 3→3², 5→5² = 1, 9, 25 → 출력: [1, 9, 25]
HARD 02 — 예외처리 코드 추적
다음 코드의 출력값은?
data = ["10", "abc", "5"] result = [] for item in data: try: result.append(int(item) * 2) except ValueError: result.append(-1) print(result)
"10"→int 성공→20, "abc"→ValueError→-1, "5"→int 성공→10 → [20, -1, 10]
HARD 03 — 출력값 직접 입력
다음 코드를 실행하면 어떤 값이 출력될까?
d = {"a": 1, "b": 2} d["c"] = 3 d["a"] = 10 print(len(d), d["a"])
딕셔너리에 키가 이미 있으면 값이 덮어씌워집니다. 키가 새로 추가되면 항목 수가 늘어납니다.
시작: {a:1, b:2} → "c" 추가: {a:1, b:2, c:3} → "a" 덮어쓰기: {a:10, b:2, c:3}
len(d) = 3, d["a"] = 10
HARD 04 — 변수 범위 (Scope)
다음 코드를 실행하면 어떤 결과가 나타날까?
x = 5 def func(): x = x + 1 return x print(func())
함수 안에서 x = x + 1처럼 대입이 있으면 파이썬은 x를 지역변수로 봅니다.
하지만 오른쪽 x + 1을 평가할 때 지역변수 x가 아직 없어서 UnboundLocalError 발생.
해결: global x를 함수 맨 위에 선언하거나, 매개변수로 넘기기.
